MySQL     5.5.16
Mac OS X  10.6
Rails     3.0.7
mysql gem 2.8.1

When I run rake to create dbs an error occured:
rake db:create --trace

** Invoke db:create (first_time)
** Invoke db:load_config (first_time)
** Invoke rails_env (first_time)
** Execute rails_env
** Execute db:load_config
** Execute db:create
rake aborted!
uninitialized constant Mysql::Error
/Users/apple/.rvm/gems/ruby-1.8.7-p352/gems/activerecord-3.0.7/lib/active_record/railties/databases.rake:64:in `create_database'

After googling it I just figured out the issue.
http://rubyizednrailified.blogspot.com/2011/03/uninitialized-constant-mysqlerror-osx.html
http://stackoverflow.com/questions/991708/rails-mysql-and-snow-leopard

gem uninstall mysql

Add the following line to .profile
export DYLD_LIBRARY_PATH="/usr/local/mysql/lib:$DYLD_LIBRARY_PATH"

env ARCHFLAGS="-arch x86_64" gem install mysql -- --with-mysql-config=/usr/local/mysql/bin/mysql_config
I'm using bundler, so I can use bundle config:
bundle config build.mysql --with-mysql-config=/usr/local/mysql/bin/mysql_config

Reinstall mysql gem:
bundle install

rake db:create

It works.
